# 前端集成测试 - 游戏生成案例

本文档用于手动测试 Yaha 游戏生成平台的核心流程。每个案例列出**前端填写内容**、**期望的 Agent 执行路径**，以及**数据库和前端期望结果**。

---

## 测试环境准备

**API 调试终端**（agent-service）：

```bash
cd services/agent-service
uv run uvicorn app.main:app --host 127.0.0.1 --port 8000 --log-level debug
```

**前端**：
- 访问 `http://localhost:3000`
- 登录后进入"创建游戏"页面
- 填写"游戏名称"和"创意描述"（对应 API 的 `title` 和 `prompt`）
- 点击"创建生成任务"按钮

### 案例 4：简单游戏 - 点击得分（统一 LLM 生成路径）

|------|------|

| 素材 | 无 |

**期望 Agent 执行路径：**

```
START
  → SupervisorAgent → [route: "approved"]
    → [并行] vision_agent / gameplay_agent / narrative_agent
    → synthesis_agent（LLM 整合设计）

```

### 案例 14：复杂游戏 - Validator 重试（验证失败 → 重试）

**前置条件：**

- 需要 `code_generator` 渲染的产物第一次验证不通过
- 触发 `should_retry` → `"retry_workflow"` → 修复 → `validator` 再次验证
- 此场景较难手动触发，通常由产物生成 bug 引发

**期望 Agent 执行路径：**

```
... → validator（第一次验证失败）
  → [should_retry 返回 "retry_workflow"]
    → retry_workflow（修复产物）
      → validator（第二次验证通过）
        → upload_workflow
          → END
```

**期望日志（终端）：**

```
Validator: 验证失败，产物结构异常
RetryWorkflow: 开始修复...
RetryWorkflow: 修复完成
Validator: 第二次验证通过
```

**期望数据库 `AgentLog`：**

| agent | step | message |
|-------|------|---------|
| Validator | validate | 验证失败 |
| RetryWorkflow | retry | 开始修复... |
| RetryWorkflow | retry_complete | 修复完成 |
| Validator | validate | 验证通过 |

**期望数据库 `GenerationTask`：`status = COMPLETED`（即使重试后仍成功）**

- 用户原始需求：设计一个塔防策略游戏，在地图上放置不同类型的防御塔来阻止敌人进攻。每种塔有不同的攻击范围、伤害和攻击速度，敌人有不同的护甲和速度。玩家需要管理金币来建造和升级塔。游戏包含 10 个波次，每波敌人数量和强度递增。击败 Boss 可获得额外金币。还有商店系统，可以购买特殊技能如空袭或冰冻陷阱。游戏采用等距视角，颜色明亮卡通风格。

## 视觉设计规范
- 视觉风格：cartoon
- 配色方案（必须严格使用）：{"primary": "#4A90E2", "accent": "#FFD700", "background": "#E0F7FA"}
- 游戏氛围：轻松愉快
- 字体风格：圆润、粗体、带有明显外描边的卡通字体，确保在复杂战场背景下清晰可读
- 动画效果要求：

  - 金币收集时的缩放弹跳与光效闪烁
  - 技能释放时的全屏震动与特效叠加

## 游戏机制规范
- 游戏类型：action
- 游戏目标：通过在等距地图上战略性地放置、升级防御塔并使用特殊技能，抵御 10 波具有不同属性的敌人进攻，保护基地免受入侵。
- 核心玩法：
  - 基于菱形网格的防御塔放置与升级系统
  - 3种基础防御塔（箭塔、炮塔、魔法塔）及其分支进化路线
  - 基于金币的经济运营与地图障碍物清理机制
  - 包含空袭、冰冻陷阱等主动技能的商店系统
  - 具有护甲与速度差异的波次敌人 AI 逻辑
- 难度曲线：由简入难，1-3波发育期，4-7波引入混合兵种，8-9波高压进攻，第10波为Boss战，难度随波次线性与阶梯式提升。
- 计分系统：基于击败敌人数量、剩余基地生命值及关卡结束时的剩余金币进行综合评分。
- 时间限制：无限制
- 获胜条件：成功抵御全部 10 波敌人并击败最终 Boss。
- 失败条件：基地生命值归零，即允许通过防线的敌人数量达到上限。
- 操作方式：鼠标点击放置防御塔、鼠标点击选择塔进行升级、鼠标点击释放特殊技能
